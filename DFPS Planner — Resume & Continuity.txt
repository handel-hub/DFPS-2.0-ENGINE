C. Plugin Indexing Layer

Status

Not implemented.


---

Problem

CP-SAT works best with integer identifiers.

Planner still carries:

plugin_id: str


---

Planned Solution

Builder stage creates:

plugin_index = {
    "ffmpeg": 1,
    "compressor": 2,
    "hashing": 3
}

Reverse map:

reverse_plugin_index = {
    1: "ffmpeg",
    2: "compressor",
    3: "hashing"
}


---

Flow

Before CP-SAT:

plugin_id -> plugin_index

After CP-SAT:

plugin_index -> plugin_id


---

Goal

Keep original planner contracts unchanged.

Perform indexing only at Builder boundary.

Final Project Position

The Planner is no longer in the architecture-design phase.

It is now in the integration, validation, and hardening phase.

The biggest remaining risks are not algorithms—they are:

1. Stage integration correctness.


2. Gateway correctness.


3. Failure/rejection handling.


4. End-to-end execution validation.



Everything else is refinement.
